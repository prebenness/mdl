# XS + S Engineering TODOs ŌĆö 2026-03-20

## What was done

Implemented all 8 XS and S engineering TODOs from the consolidated code-cleanup list.

### XS items

**1. Entropy variable naming** ŌĆö Updated docstrings and comments in `training.py`, `shared_weights.py`, `steps.py`, and `differentiable_mdl.py` to consistently use "entropy bonus (subtracted): tau * H(pi)" instead of the proposal's "-(1/beta) * H(pi)" notation.

**2. Short-circuit `n_samples` for `mode_forward`** ŌĆö When `mode_forward=True`, the forward pass is deterministic, so multi-sampling wastes compute. Added `n_samples = 1 if cfg.mdl.mode_forward else cfg.mdl.n_samples` at all 4 cMNIST step factories in `steps.py`, and `effective_n_samples` guards at both ANBN train-step creation sites in `differentiable_mdl.py`.

**3. Vectorize `_compute_discrete_hyp_bits`** ŌĆö Replaced the Python `for` loop + per-element `rational_codelength()` call with a single `jnp.sum(grid_codelengths[idx])` lookup. Updated all 4 call sites to pass `grid_codelengths` instead of `grid, grid_values`.

### S items

**4. Full fixed test sets during evaluation** ŌĆö Added `make_eval_batches(x, y, batch_size)` in `datasets.py` that pads the last batch and returns a per-batch `counts` array. Updated `make_eval_epoch` in `epochs.py` to weight loss/accuracy by counts. Updated all 4 cMNIST runners to create eval batches once before the training loop.

**5. Train tail-batch handling** ŌĆö Documented in `make_epoch_batches` docstring: random permutation means different samples are dropped each epoch, so all samples are seen over multiple epochs. Acceptable for training; no code change needed.

**6. Clarify `p_test` for `cmnist_ula`** ŌĆö Added a `warnings.warn()` in `build_dataset` when `name == "cmnist_ula"` and `p_test != 0.1`, since `CMNISTuLA` always uses unbiased (uniform) test colors per the uLA protocol.

**7. Enforce `rep_dim` for ULA** ŌĆö `ULAMLPClassifier` now emits a warning if `rep_dim != 100`. Hidden widths remain fixed at 100 for strict fidelity to the papers being replicated.

**8. Unify batch scaling convention** ŌĆö Changed ANBN `_compute_data_nll_bits` and `_shared_compute_data_nll_bits` to return averaged NLL (divided by valid mask count) instead of summed. Changed `batch_scale = B / n_train` ŌåÆ `hyp_scale = 1.0 / n_train` in both `make_loss_fn` (basic) and `make_shared_loss_fn` (shared). Updated `mdl-metrics-and-logging.MD` formulas. **Important: existing ANBN hyperparameters (lambda, lr) will need recalibration.**
